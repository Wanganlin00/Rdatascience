11.1 基础知识

  • 每个环境名称都必须是唯一的。

  • 环境中的名称不按顺序排列。

  • 环境有其父级。

  • 修改时不会复制环境。

11.1.1 创建环境

11.1.3 父环境

每个环境都有一个父环境（parent）。父级是用于实现词法作用域的：如果在环境中找不到名称，则 R 将在其父级中查找（依此类推）。可以通过向 env() 提供未命名的参数来设置父环境。如果不提供它，它将默认为当前环境。

Code
e2a <- env(d = 4, e = 5)
e2b <- env(e2a, a = 1, b = 2, c = 3)

e2a 是 e2b 的父级。

Code
# 查看父级
e2a
#> <environment: 0x00000191ca5ac458>
env_parent(e2b)
#> <environment: 0x00000191ca5ac458>
env_parent(e2a)
#> <environment: R_GlobalEnv>

只有一个环境没有父环境：空( empty)环境。

Code
e2c <- env(empty_env(), d = 4, e = 5)
e2d <- env(e2c, a = 1, b = 2, c = 3)

每个环境的父级最终都会随着空环境而终止。

Code
# 查看所有父级

#> [[1]]   <env: 0x00000191c716bbc8>
#> [[2]] $ <env: empty>
e2c
#> <environment: 0x00000191c716bbc8>
env_parents(e2c)
#> [[1]] $ <env: empty>

11.2 在环境中递归（recurse）

11.2.1 递归

Code
f_recur <- function(..., env = caller_env()) {
  if (identical(env, empty_env())) {
    # base case
  } else if (success) {
    # success case
  } else {
    # recursive case
    f(..., env = env_parent(env))
  }
}
Code
where <- function(name, env = caller_env()) {
  if (identical(env, empty_env())) {
    # Base case
    stop("Can't find ", name, call. = FALSE)
  } else if (env_has(env, name)) {
    # Success case
    env
  } else {
    # Recursive case
    where(name, env_parent(env))
  }
}
Code
e3a <- env(empty_env(), a = 1, b = 2)
e3b <- env(e3a, x = 10, a = 11)
Code
where("a", e3b)
#> <environment: 0x00000191c9bbec88>

where("b", e3b)
#> <environment: 0x00000191c9c69420>
e3a
#> <environment: 0x00000191c9c69420>

where("c", e3b)
#> Error: Can't find c

11.2.2 迭代iteration

Code
f_iter <- function(..., env = caller_env()) {
  while (!identical(env, empty_env())) {
    if (success) {
      # success case
      return()
    }
    # inspect parent
    env <- env_parent(env)
  }

  # base case
}

11.3.2 函数环境

函数在创建时绑定当前环境，称为函数环境，用于词法作用域。在计算机语言中，捕获（或封闭）其环境的函数被称为闭包 closures 。

访问函数环境

Code
y <- 1
f <- function(x) x + y
fn_env(f)
#> <environment: R_GlobalEnv>
Code
e <- env()  # 绑定当前环境为父级
env_parent(e)
#> <environment: R_GlobalEnv>

e$g <- function() 1  # g绑定在e中

11.3.3 命名空间

命名空间 （namespaces） 的目标是确保以不同的顺序加载包，包将找到相同的功能，并且无论用户附加什么包，每个包的工作方式都相同。

Code
sd
#> function (x, na.rm = FALSE) 
#> sqrt(var(if (is.vector(x) || is.factor(x)) x else as.double(x), 
#>     na.rm = na.rm))
#> <bytecode: 0x00000191d043cca8>
#> <environment: namespace:stats>

包中的每个函数都与一对环境相关联：前面的包环境和命名空间环境。

  • 包环境是包的外部接口。
  • 命名空间环境是包的内部接口。

包环境控制我们如何找到函数;命名空间控制函数查找其变量的方式。

包环境中的每个绑定也可以在命名空间环境中找到。

每个命名空间都有一个import环境，import环境的父级是base namespace，基本命名空间的父级是全局环境。

11.3.4 执行环境

执行环境是函数环境的子级，由函数的创建位置决定。

Code
g <- function(x) {
  if (!env_has(current_env(), "a")) {
    message("Defining a")
    a <- 1
  } else {
    a <- a + 1
  }
  a
}
Code
# A fresh start principle
g(10)
#> [1] 1
g(10)
#> [1] 1

每次调用函数时，都会创建一个新环境来托管执行。这称为执行环境（execution environment），其父级是函数环境 Figure 11.1 。

Code
h <- function(x) {
  # 1.
  a <- 2 # 2.
  x + a
}
y <- h(1) # 3.

执行环境通常是短暂的;函数完成后，环境将被垃圾回收（garbage collected）。

11.4 Call stacks

caller environment

rlang::caller_env() 提供了调用函数的环境。

调用栈（call stacks）由 frames 组成。调用栈是由调用函数的位置创建的。

Code
f <- function(x) {
  g(x = 2)
}
g <- function(x) {
  h(x = 3)
}
h <- function(x) {
  stop()
}

在 R 中最常查看调用栈的方式是查看错误发生后的情况：traceback()

Code
f(x = 1)
#> Error in h(x = 3):
traceback()
#> No traceback available

使用 lobstr::cst()从头开始 来理解调用栈, call stack tree Figure 11.2

Code
h <- function(x) {
  lobstr::cst()
}
f(x = 1)
#>     ▆
#>  1. └─global f(x = 1)
#>  2.   └─global g(x = 2)
#>  3.     └─global h(x = 3)
#>  4.       └─lobstr::cst()

11.4.1 frames

调用栈的每个元素都是一个frame，也称为 evaluation context。Frames 是一个极其重要的内部数据结构，R 代码只能访问它的一小部分，因为篡改它会破坏 R。

frame 有三个关键组件：

  • 给出函数调用的表达式（标有expr ）
  • 环境（标记为env ）
  • 父级，调用栈中的上一个调用（由灰色箭头显示）

在调用栈中而不是在封闭环境中查找变量称为dynamic scoping 。

### 函数环境

函数在创建时绑定当前环境，称为**函数环境**，用于词法作用域。在计算机语言中，捕获（或封闭）其环境的函数被称为闭包 closures 。

访问函数环境

```{r}
y <- 1
f <- function(x) x + y
fn_env(f)
```

![](images/function_environment.png){fig-align="center" width="30%"}

```{r}
e <- env()  # 绑定当前环境为父级
env_parent(e)

e$g <- function() 1  # g绑定在e中
```

### 命名空间

**命名空间** （namespaces） 的目标是确保以不同的顺序加载包，包将找到相同的功能，并且无论用户附加什么包，每个包的工作方式都相同。

```{r}
sd
```

包中的每个函数都与一对环境相关联：前面的包环境和命名空间环境。

-   包环境是包的外部接口。
-   命名空间环境是包的内部接口。

包环境控制我们如何找到函数;命名空间控制函数查找其变量的方式。

包环境中的每个绑定也可以在命名空间环境中找到。

每个命名空间都有一个**import**环境，**import**环境的父级是**base namespace**，基本命名空间的父级是全局环境。

![](images/namespace.png){fig-align="center" width="50%"}

### 执行环境

执行环境是函数环境的子级，由函数的创建位置决定。

```{r}
g <- function(x) {
  if (!env_has(current_env(), "a")) {
    message("Defining a")
    a <- 1
  } else {
    a <- a + 1
  }
  a
}

```

```{r}
# A fresh start principle
g(10)
g(10)
```

每次调用函数时，都会创建一个新环境来托管执行。这称为执行环境（**execution environment**），其父级是函数环境 @fig-execution 。

```{r}
h <- function(x) {
  # 1.
  a <- 2 # 2.
  x + a
}
y <- h(1) # 3.
```

![](images/execution.png){#fig-execution fig-align="center" width="35%"}

执行环境通常是短暂的;函数完成后，环境将被垃圾回收（*garbage collected*）。

## Call stacks

caller environment

`rlang::caller_env()` 提供了调用函数的环境。

调用栈（**call stacks**）由 **frames** 组成。调用栈是由调用函数的位置创建的。

```{r}
f <- function(x) {
  g(x = 2)
}
g <- function(x) {
  h(x = 3)
}
h <- function(x) {
  stop()
}
```

在 R 中最常查看调用栈的方式是查看错误发生后的情况：traceback()

```{r error=TRUE}
f(x = 1)
traceback()
```

使用 `lobstr::cst()`从头开始 来理解调用栈, **c**all **s**tack **t**ree @fig-frame

```{r}
h <- function(x) {
  lobstr::cst()
}
f(x = 1)
```

### frames

调用栈的每个元素都是一个**frame**，也称为 **evaluation context**。Frames 是一个极其重要的内部数据结构，R 代码只能访问它的一小部分，因为篡改它会破坏 R。

frame 有三个关键组件：

-   给出函数调用的表达式（标有`expr` ）
-   环境（标记为`env` ）
-   父级，调用栈中的上一个调用（由灰色箭头显示）

![](images/calling_stack_frame.png){#fig-frame fig-align="center" width="45%"}
