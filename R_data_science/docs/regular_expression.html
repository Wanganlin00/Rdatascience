10.2 量词 Quantifier

10.2.1 贪婪量词

尽可能匹配

n*      匹配任意次数（0 or more than n）       {0,}
n+      至少匹配1次（1 or more than n ）       {1,}
n?      匹配 0或1次（0 or 1 n）                {0,1}

{n,}    至少匹配 n 次。
{n,m}   N ~ M 次之间的匹配。
{n}     正好匹配 n 次。
Show the code
x <- c("1234_abcd abbb", "123 a33aabb", " ab.bcaab", "abbb_132bb")

str_view(x, "ab*") 
#> [1] │ 1234_<ab>cd <abbb>
#> [2] │ 123 <a>33<a><abb>
#> [3] │  <ab>.bc<a><ab>
#> [4] │ <abbb>_132bb

str_view(x, "ab+") 
#> [1] │ 1234_<ab>cd <abbb>
#> [2] │ 123 a33a<abb>
#> [3] │  <ab>.bca<ab>
#> [4] │ <abbb>_132bb

str_view(x, "ab?")
#> [1] │ 1234_<ab>cd <ab>bb
#> [2] │ 123 <a>33<a><ab>b
#> [3] │  <ab>.bc<a><ab>
#> [4] │ <ab>bb_132bb

str_view(x, "b{2,5}")
#> [1] │ 1234_abcd a<bbb>
#> [2] │ 123 a33aa<bb>
#> [4] │ a<bbb>_132<bb>
str_view(x, "\\d{2}")
#> [1] │ <12><34>_abcd abbb
#> [2] │ <12>3 a<33>aabb
#> [4] │ abbb_<13>2bb
str_view(x, "\\d{2,}")
#> [1] │ <1234>_abcd abbb
#> [2] │ <123> a<33>aabb
#> [4] │ abbb_<132>bb
str_view(x, "\\d{2,3}")
#> [1] │ <123>4_abcd abbb
#> [2] │ <123> a<33>aabb
#> [4] │ abbb_<132>bb

10.2.2 懒惰量词

在满足条件的情况下会尽量少地匹配

原有量词后加一个 ? ，仅保持最小匹配

Show the code
str_view(x, "b{3,5}?")  #匹配bbb
#> [1] │ 1234_abcd a<bbb>
#> [4] │ a<bbb>_132bb
str_view(x, "abb??")  # 匹配ab
#> [1] │ 1234_<ab>cd <ab>bb
#> [2] │ 123 a33a<ab>b
#> [3] │  <ab>.bca<ab>
#> [4] │ <ab>bb_132bb
str_view(x, "ab+?")  # 匹配ab
#> [1] │ 1234_<ab>cd <ab>bb
#> [2] │ 123 a33a<ab>b
#> [3] │  <ab>.bca<ab>
#> [4] │ <ab>bb_132bb
str_view(x, "ab*?")  # 匹配a
#> [1] │ 1234_<a>bcd <a>bbb
#> [2] │ 123 <a>33<a><a>bb
#> [3] │  <a>b.bc<a><a>b
#> [4] │ <a>bbb_132bb

10.3.3 Unicode字符

  • 基本 Unicode 范围匹配：对于支持 Unicode 的正则表达式引擎，可以匹配更广泛的字符范围。例如，\p{L}匹配任何语言中的字母字符。\p{Nd}匹配任何十进制数字（相当于[0 - 9]的 Unicode 版本）。

    Show the code
    x <- c("拼音aāà⇌α", "拉丁aáǎβ", "a€∀ζ","货币123¥δ")

str_view(x,pattern = "\\p{L}")
#> [1] │ <拼><音><a><ā><à>⇌<α>
#> [2] │ <拉><丁><a><á><ǎ><β>
#> [3] │ <a>€∀<ζ>
#> [4] │ <货><币>123¥<δ>
str_view(x,pattern = "\\p{Nd}")
#> [4] │ 货币<1><2><3>¥δ

  • 特定语言或脚本的 Unicode 范围：可以使用更具体的 Unicode 脚本块来匹配字符。比如，\p{Han}匹配中文、日文、韩文等汉字字符。\p{Greek}匹配希腊字母。这些对于处理多语言文本非常有用，例如在文本提取或格式化中，根据不同语言的字符进行区分。

Show the code
str_view(x,pattern = "\\p{Han}")
#> [1] │ <拼><音>aāà⇌α
#> [2] │ <拉><丁>aáǎβ
#> [4] │ <货><币>123¥δ
str_view(x,pattern = "\\p{Greek}")
#> [1] │ 拼音aāà⇌<α>
#> [2] │ 拉丁aáǎ<β>
#> [3] │ a€∀<ζ>
#> [4] │ 货币123¥<δ>

10.6 零宽匹配

断言（assertions）是一种零宽度（zero - width）的匹配，它用于指定一个位置，这个位置应该满足某种条件，但不会消耗（匹配）任何字符。

zero-width match

Show the code
str_view("abc", c("$", "^", "\\b"))
#> [1] │ abc<>
#> [2] │ <>abc
#> [3] │ <>abc<>
str_replace_all(
    string = "abc",
    pattern =  c("$", "^", "\\b"),
    replacement =  "+_+"
)
#> [1] "abc+_+"    "+_+abc"    "+_+abc+_+"

10.6.1 断言assertions

  • (?=pattern) 要求此位置的后面必须匹配表达式pattern

  • (?!pattern) 要求此位置的后面不能匹配表达式pattern

  • (?<=pattern) 要求此位置的前面必须匹配表达式pattern

  • (?<!pattern) 要求此位置的前面不能匹配表达式pattern

Show the code
x <- c("ocauuno", "0on0242uh","nauguio","nucu0ono0huhun")

# 正向肯定预查（Positive Lookahead）
str_view(x,pattern = "n(?=o)")  # 向后匹配 n后面紧跟着o，不匹配o
#> [1] │ ocauu<n>o
#> [4] │ nucu0o<n>o0huhun

# 正向否定预查（Negative Lookahead）
str_view(x,pattern = "n(?!o)")  # 匹配后面不跟着o的n
#> [2] │ 0o<n>0242uh
#> [3] │ <n>auguio
#> [4] │ <n>ucu0ono0huhu<n>



# 反向肯定预查（Positive Lookbehind）
str_view(x,pattern = "(?<=0o)n")  # 匹配前面是0o的n
#> [2] │ 0o<n>0242uh
#> [4] │ nucu0o<n>o0huhun

# 反向否定预查（Negative Lookbehind）

str_view(x,pattern = "(?<!0o)n")  # 匹配前面不是0o的n
#> [1] │ ocauu<n>o
#> [3] │ <n>auguio
#> [4] │ <n>ucu0ono0huhu<n>

10.7 反向引用backreference

反向引用（Backreference）允许引用之前在正则表达式中匹配到的子表达式。简单来说，就是可以在正则表达式的后续部分重复使用之前已经匹配的内容。它通过使用\数字的形式来实现，其中 “数字” 表示之前捕获组的编号。

Show the code
# 有哪些字母是重复两次的
str_view(fruit, "(.)\\1", match = TRUE)
#>  [1] │ a<pp>le
#>  [5] │ be<ll> pe<pp>er
#>  [6] │ bilbe<rr>y
#>  [7] │ blackbe<rr>y
#>  [8] │ blackcu<rr>ant
#>  [9] │ bl<oo>d orange
#> [10] │ bluebe<rr>y
#> [11] │ boysenbe<rr>y
#> [16] │ che<rr>y
#> [17] │ chili pe<pp>er
#> [19] │ cloudbe<rr>y
#> [21] │ cranbe<rr>y
#> [23] │ cu<rr>ant
#> [28] │ e<gg>plant
#> [29] │ elderbe<rr>y
#> [32] │ goji be<rr>y
#> [33] │ g<oo>sebe<rr>y
#> [38] │ hucklebe<rr>y
#> [47] │ lych<ee>
#> [50] │ mulbe<rr>y
#> ... and 9 more

str_view(fruit, "(..)\\1", match = TRUE)
#>  [4] │ b<anan>a
#> [20] │ <coco>nut
#> [22] │ <cucu>mber
#> [41] │ <juju>be
#> [56] │ <papa>ya
#> [73] │ s<alal> berry

捕获组是通过在正则表达式中使用括号()来定义的。例如，在正则表达式()-()中，()是第一个捕获组，()是第二个捕获组。捕获组的编号是从左到右，从 1 开始计数。当这个正则表达式匹配一个电话号码，如123 - 4567时，123被存储在第一个捕获组中，4567被存储在第二个捕获组中。

Show the code
str_replace(string = c("123 - 4567 - 0030 - 789",
                           "123 - 4567 - 4567 - 123"),
         pattern = "(\\d{3}) - (\\d{4}) - \\2 - \\1",
         replacement = "backreference")
#> [1] "123 - 4567 - 0030 - 789" "backreference"

有时候，我们可能不希望捕获组存储匹配内容，但又想使用分组来控制优先级或者方便使用反向引用。这时候可以使用非捕获组，其语法是(?:…)。例如，在正则表达式(?:)-()中，(?:)是一个非捕获组，它匹配 3 位数字但不存储，只有()是捕获组。如果要在后续部分引用部分的内容，就不能使用反向引用，因为它没有被捕获。

Show the code
str_replace(string = c("123 - 4567 - 0030 - 789",
                           "123 - 4567 - 45657 - 123"),
         pattern = "(\\d{3}) - (?:\\d{4}) - (?:\\d{5}) - \\1",
         replacement = "no backreference")
#> [1] "123 - 4567 - 0030 - 789" "no backreference"

## 量词 Quantifier

### 贪婪量词

尽可能匹配

```         
n*      匹配任意次数（0 or more than n）       {0,}
n+      至少匹配1次（1 or more than n ）       {1,}
n?      匹配 0或1次（0 or 1 n）                {0,1}

{n,}    至少匹配 n 次。
{n,m}   N ~ M 次之间的匹配。
{n}     正好匹配 n 次。
```

```{r}
x <- c("1234_abcd abbb", "123 a33aabb", " ab.bcaab", "abbb_132bb")

str_view(x, "ab*") 

str_view(x, "ab+") 

str_view(x, "ab?")

str_view(x, "b{2,5}")
str_view(x, "\\d{2}")
str_view(x, "\\d{2,}")
str_view(x, "\\d{2,3}")
```

### 懒惰量词

在满足条件的情况下会尽量少地匹配

原有量词后加一个 ? ，仅保持最小匹配

```{r}
str_view(x, "b{3,5}?")  #匹配bbb
str_view(x, "abb??")  # 匹配ab
str_view(x, "ab+?")  # 匹配ab
str_view(x, "ab*?")  # 匹配a

```

## 零宽匹配

断言（assertions）是一种零宽度（zero - width）的匹配，它用于指定一个位置，这个位置应该满足某种条件，但不会消耗（匹配）任何字符。

zero-width match

```{r}
str_view("abc", c("$", "^", "\\b"))
str_replace_all(
    string = "abc",
    pattern =  c("$", "^", "\\b"),

```

### 断言assertions

-   `(?=pattern)` 要求此位置的后面必须匹配表达式pattern

-   `(?!pattern)` 要求此位置的后面不能匹配表达式pattern

-   `(?<=pattern)` 要求此位置的前面必须匹配表达式pattern

-   `(?<!pattern)` 要求此位置的前面不能匹配表达式pattern

```{r}
x <- c("ocauuno", "0on0242uh","nauguio","nucu0ono0huhun")

# 正向肯定预查（Positive Lookahead）
str_view(x,pattern = "n(?=o)")  # 向后匹配 n后面紧跟着o，不匹配o

# 正向否定预查（Negative Lookahead）
str_view(x,pattern = "n(?!o)")  # 匹配后面不跟着o的n



# 反向肯定预查（Positive Lookbehind）
str_view(x,pattern = "(?<=0o)n")  # 匹配前面是0o的n

# 反向否定预查（Negative Lookbehind）

str_view(x,pattern = "(?<!0o)n")  # 匹配前面不是0o的n
```

## 反向引用backreference

反向引用（Backreference）允许引用之前在正则表达式中匹配到的子表达式。简单来说，就是可以在正则表达式的后续部分重复使用之前已经匹配的内容。它通过使用`\数字`的形式来实现，其中 “数字” 表示之前捕获组的编号。

```{r}
# 有哪些字母是重复两次的
str_view(fruit, "(.)\\1", match = TRUE)

str_view(fruit, "(..)\\1", match = TRUE)
```

捕获组是通过在正则表达式中使用括号()来定义的。例如，在正则表达式(\d{3})-(\d{4})中，(\d{3})是第一个捕获组，(\d{4})是第二个捕获组。捕获组的编号是从左到右，从 1 开始计数。当这个正则表达式匹配一个电话号码，如123 - 4567时，123被存储在第一个捕获组中，4567被存储在第二个捕获组中。

```{r}
str_replace(string = c("123 - 4567 - 0030 - 789",
                           "123 - 4567 - 4567 - 123"),
         pattern = "(\\d{3}) - (\\d{4}) - \\2 - \\1",
         replacement = "backreference")
```

有时候，我们可能不希望捕获组存储匹配内容，但又想使用分组来控制优先级或者方便使用反向引用。这时候可以使用非捕获组，其语法是(?:...)。例如，在正则表达式(?:\d{3})-(\d{4})中，(?:\d{3})是一个非捕获组，它匹配 3 位数字但不存储，只有(\d{4})是捕获组。如果要在后续部分引用\d{3}部分的内容，就不能使用反向引用，因为它没有被捕获。

```{r}
str_replace(string = c("123 - 4567 - 0030 - 789",
                           "123 - 4567 - 45657 - 123"),
         pattern = "(\\d{3}) - (?:\\d{4}) - (?:\\d{5}) - \\1",
         replacement = "no backreference")
```
